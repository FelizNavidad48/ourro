## Context

The roadmap is functionally complete: an independent audit confirmed every
M1тАУM5 milestone is genuinely implemented, wired, and covered by the 552-check
FiveAM suite, with essentially zero dead code. The remaining distance to the
north star тАФ a Lisp-native evolving agent with minimal friction and a TUI
whose UX makes evolution obvious тАФ is concentrated in four places:

1. **Hygiene debt** тАФ M0 was never executed: 11.5 MB `bin/ourro` + 18 `.fasl`
   files still git-tracked; `.gitignore` is the broken one-liner `!bin/*`;
   the four doc deletions in the working tree are unstaged; README/ROADMAP
   scorecard rows are stale.
2. **Small defects in `fable-3-m5`** (otherwise a high-quality commit тАФ verdict
   below).
3. **TUI/UX gaps** that are live-demo risks: no turn-cancel (ctrl-c quits the
   whole app), no always-on proof that evolution pays off, undiscoverable
   marquee keys, char-count width math (emoji/CJK shear the layout), no
   mouse/scroll affordances, tool output invisible past its first line.
4. **Stale skill/docs** тАФ SKILL.md predates the ledger/corrections/queue/
   inspector/onboard/markdown/determinism subsystems.

User decisions (asked & answered): TUI/UX cockpit before adaptation depth;
genome sharing **out of scope** this cycle; a **live end-to-end verification
pass** (gcloud auth, real supervised run) **in scope**.

## Verdict on commit `44345ba` (fable-3-m5)

**Keep it тАФ it clearly benefits the project.** It closes the last three
scorecard rows (PR-5 restart budget measured, PR-12 rebuild-on-demand from the
genome git commit, PR-13 determinism as walker-enforced structure + a
gauntlet-verified contract), reuses existing seams (`*build-image-hook*`, the
build lock, `verify-determinism`, staged registries) instead of reinventing,
and its tests are substantive тАФ the gensym-probe test proves the determinism
stage adds coverage beyond a gene's own tests; the rebuild test runs real
`git worktree` against a scratch genome. Confirmed defects, fixed in M6:

- **D1 (low-med)** `src/supervisor.lisp:843-844` тАФ the boot/crash pre-spawn
  guard wraps `ensure-generation-image` in `ignore-errors` and **discards the
  return value**; a missing image whose rebuild fails still gets exec'd тЖТ the
  unhandled `launch-program` error crashes the whole supervisor. (The handoff
  branch at `:887-892` checks correctly.)
- **D2 (low)** тАФ orphaned `state/worktrees/<id>/` dirs from an interrupted
  rebuild are only cleaned when that same id is rebuilt; nothing GCs the dir.
- **D3 (info)** тАФ determinism probes run under bare gene caps while staged
  tests get `union(caps, fs-read/write)`. Intentional-in-effect (the walker
  guarantees a tool can't need undeclared caps; the test-stage extras exist
  for FiveAM fixtures) тАФ **document, don't change**.
- **D4 (info)** тАФ `restart-timer` unlocked cross-thread access (single-writer
  publish, torn read costs one measurement; matches the `agent-said-hello`
  pattern) тАФ comment only.
- **D5 (info)** тАФ generation tags are never deleted (harmless; document).

### M7-1 ┬╖ тЬЕ Turn-cancel тАФ conditions/restarts on camera

Today ctrl-c/ctrl-d hard-quit (`agent.lisp:1433-1434`); a 25-iteration tool
loop cannot be stopped. Design: a cancellation **flag** checked at the turn's
natural boundaries, signaled as a **non-error `serious-condition`**
`turn-cancelled` that unwinds the worker untouched through every generic
`(error тАж)` handler тАФ verified: the vertex chunk-guard at
`src/llm/vertex.lisp:255` is `(handler-case тАж (error () nil))`, and
`complete-with-retry` only handles `provider-error`, so the cancel passes
through with **zero vertex.lisp changes**. Escalation via
`bt:interrupt-thread` for a worker stuck in a blocking read, made safe by a
thread-local `*cancel-inhibited*` binding around uninterruptible sections.

1. **Condition + special** (`src/kernel/conditions.lisp`, both exported):
   `(define-condition turn-cancelled (serious-condition) ((reason тАж))
   (:documentation "Deliberately NOT an ERROR subtype тАж"))` and
   `(defvar *cancel-inhibited* nil)`.
2. **Agent slots** (`src/agent.lisp` ~line 54): `cancel-requested`,
   `cancel-time`, `cancel-escalated`, `turn-thread`, `last-interrupt` тАФ plain
   single-writer slots, same discipline as `busy`.
3. **Helpers**: `request-cancel` (UI thread: set flag + `set-activity
   "cancellingтАж (ctrl-c again to quit)"`); pure `interrupt-action (busy-p
   last-press now &key window-ms)` тЖТ `:quit` when idle or double-pressed
   within 1.5 s, else `:cancel`; `check-cancel` тЖТ `(error 'turn-cancelled)`
   when flagged.
4. **Key routing** (`handle-editor-key`): `:escape` with busy + empty input тЖТ
   `request-cancel`, else clear input as today; `:ctrl-c` тЖТ `interrupt-action`
   dispatch (cancel first press, quit on double-press or when idle);
   `:ctrl-d` always quits.
5. **Flag checks on the turn worker** (the sanctioned transcript writer, D-1):
   first form of `stream-event` (`:435`); top of `process-turn`'s `dotimes`
   body (`:318`); in `run-tool-calls` (`:380`) before each call тАФ and when
   flagged, **synthesize a cancelled `tool-result-message` (`:error-p t`) for
   every remaining call** so the Gemini conversation stays well-formed (every
   `functionCall` needs a `functionResponse`, else the next turn 400s), plus a
   dim `"тП╣ <name> skipped (cancelled)"` transcript line.
6. **Unwind handler**: wrap `process-turn`'s `dotimes` in
   `(handler-case тАж (turn-cancelled () тАж))` тАФ finalize partial stream text via
   `finish-stream` (keep what streamed), `reset-stream`, dim `"тП╣ (cancelled)"`
   line, `log-event :turn-cancelled`, clear activity. The existing
   `unwind-protect` cleanup still clears `busy` and fires `:turn-done`
   (checkpoint/mining/typeahead drain proceed normally). Clear
   `cancel-requested` in that cleanup **and** at claim time in
   `run-submission` (both branches, next to `(setf (agent-busy agent) t)`,
   `:1581/:1596`); record `turn-thread` there.
7. **Escalation** (`maybe-escalate-cancel`, called from `ui-loop` after
   `maybe-dream`, `:1689`): 2 s after a request with the worker still busy тЖТ
   `bt:interrupt-thread` with a lambda that signals `turn-cancelled` **unless
   the target thread's `*cancel-inhibited*` is bound true**. Bind it around
   `deliberate-evolution`'s body and `hot-load-gene`'s body. Belt-and-braces:
   a `turn-cancelled` clause in `run-submission`'s worker `handler-case`
   (`:1600-1603`) so an interrupt landing outside `process-turn` kills only
   the turn.
8. **Subprocesses**: not killed in v1 тАФ an escalated interrupt unwinds the
   Lisp side; document (future: live-subprocess registry in `cap/*`).

**Tests** (new `tests/cancel-test.lisp`, registered in `ourro.asd`):
`turn-cancelled-is-not-an-error` (pins the transparency property with a nested
`handler-case` тАФ the load-bearing invariant); `cancel-mid-stream-finalizes-
partial-text` (scripted `:stream t` provider; no partial assistant message in
